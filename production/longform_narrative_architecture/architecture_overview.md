# Architecture Overview

## 总链路

```text
IP 宇宙
  -> 世界 / 子世界
  -> 单书核心故事
  -> 主线 / 分支线
  -> 卷结构
  -> 章节链
  -> 场景 / 冲突单元
  -> 角色主动感发动机
  -> 正文渲染
  -> 状态回写
  -> 下一章继续
```

## “车”的类比

- IP 宇宙：燃料 / 世界资源。
- 单书故事：目的地。
- 卷结构：阶段路线。
- 章节链：导航点。
- 角色主动感：发动机。
- 人物质感：燃烧质感。
- 作品声音：驾驶手感。
- 读者代入评价：乘客体验。
- 状态回写：里程表和导航修正。

## 核心原则

车架负责让长篇稳定运行，但发动机负责让场景真正动起来。不能因为车架设计完整，就把人物变成章节导航点上的执行件。
