# Polisher Boundary Design

Polisher already has a Phase 8 position: light enhancement, not rescue. Work Voice injection should reinforce that boundary rather than expand Polisher authority.

## Polisher May Do

- Smooth local sentences.
- Delete over-explanation.
- Tighten rhythm.
- Reduce slogan-like phrasing.
- Reduce overly stiff or old-fashioned tone when structure already passes.
- Correct local tone drift that does not affect narrator position.
- Improve sentence rhythm after Reviewer confirms Work Voice structure is valid.

## Polisher Must Not Do

- Add missing author stance.
- Rebuild `narrator_position`.
- Rewrite `reader_relationship`.
- Fix vague `voice_contract`.
- Fix Planner scene_type selection errors.
- Fix Writer output that became specific author imitation.
- Supply a full narrator personality after the draft lacks one.
- Add catchphrases, formulaic lines, or imitation syntax to make text “more stylish”.
- Change beat events, result, causality, protagonist motive, relationship outcome, or canon.

## Routing Decision

| Problem | Route |
|---|---|
| Local sentence roughness | Polisher allowed |
| Repeated explanatory phrases but stance is stable | Polisher allowed with narrow instruction |
| Chapter does not know who is telling it | Return to Work Voice Contract / Planner / Writer |
| Reader relationship absent | Return to Work Voice Contract |
| Scene stance wrong for payoff or hook | Return to Planner / Contract |
| Writer copied source-specific flavor | Fail, return to Contract and contamination cleanup |
| Human Texture became random life details | Return to Writer / Planner |

## Key Rule

If the problem is “local sentences are rough,” Polisher may help. If the problem is “this chapter has no stable storyteller,” it must not go to Polisher.
