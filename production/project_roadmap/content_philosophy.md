# 内容价值观与反低质投喂原则

## 1. 项目为什么需要内容价值观

技术路线越强，越容易滑向“更高效地产出套路”。一个足够懂网文、懂钩子、懂平台反馈的系统，如果没有上位约束，很容易从“提升创作质量”滑向“批量制造注意力饲料”。

数据驱动可以发现钩子，但不能替代创作判断。它能告诉我们什么更容易让人停留、点击、反馈，却不能告诉我们什么值得写、什么值得被读者认真投入。

网文需要爽点、节奏、钩子。它们是入口，是阅读发动机的一部分，但不是作品灵魂。真正让读者留下来的，不只是被勾住的生理反应，而是人物的活性、叙述的诚意、压力释放的空间，以及某种“像人在现场认真写”的感觉。

因此，内容价值观不是文学宣言，而是工程约束。它要求每个技术模块都回答一个问题：它是在增强作品本身，还是只是在增强操控读者的能力？

## 2. 平台逻辑的风险

平台最擅长优化停留、点击、反馈和分发。它可以快速捕捉用户的生理反应，也能把有效刺激转化成可复制的内容模板。

风险在于，“能勾住人”不等于“值得写”。低质内容农场会利用这种机制批量投喂低质内容，让读者的注意力被污染，让创作者误以为只要命中刺激就等于作品成立。

本项目不能把平台逻辑当成最终目标。平台反馈可以作为观察材料，但不能成为创作价值观。数据可以提醒我们入口在哪里，却不能替我们决定作品要把读者带到哪里。

## 3. 喂屎、预制菜、工程化厨房、现场手艺

| 内容生产状态 | 特征 | 优点 | 风险 | 本项目态度 |
| --- | --- | --- | --- | --- |
| 喂屎 | 低质批量投喂，只要点击，不管污染 | 短期反馈快 | 污染注意力，消耗读者信任 | 明确拒绝 |
| 差预制菜 | AI 批量生成加提示词约束，有格式但没有真诚 | 便宜、快速、有基本外形 | 套路感重，人物空，现场感低 | 不作为目标 |
| 好预制菜 | 框架化生产加稳定质量控制，能吃、量大、但现场感有限 | 稳定、可控、能交付 | 上限有限，容易变成安全模板 | 可借鉴流程，不满足于此 |
| 工程化厨房 | 有流程、有标准、有备料，但努力保留现场感和创作判断 | 兼顾产能、质量和现场调整 | 需要持续校准，不能只看指标 | 本项目定位 |
| 现场手艺 | 大神作者古法写作，高上限 | 真诚、灵活、个人风格强 | 成本高、产能不稳定、不可复制 | 尊重但不伪装成它 |

本项目不做喂屎机器，也不满足于低级预制菜。本项目目标是工程化厨房：用工程化保证产能，用角色驱动保证活性，用叙述控制保证可读，用反套路原则防止滑向低质投喂。

## 4. 工业化真诚

工业化真诚 = 承认网文需要工程化、规模化、稳定产出，但工程化不是为了更高效地套路读者，而是为了尽量稳定地产出“像人认真写过”的内容。

真诚不是不能工程化。工程化也不天然等于套路。关键在于工程化服务什么目标。

如果工程化只服务点击最大化，就会滑向喂屎。如果工程化服务人物活性、叙述节制、读者参与和现场感，就可能成为高质量内容生产能力。

工业化真诚的目标不是假装机器没有参与，而是让机器参与的部分服务于更稳定的创作判断：减少断裂、减少模板、减少空心爽点，让人物、叙述者和读者空间都能被看见。

## 5. 各模块如何避免滑向套路

| 模块 | 错误方向 | 正确方向 |
| --- | --- | --- |
| 爽点机制 | 把爽点密度当成作品质量，只追求刺激堆叠 | 爽点服务人物压力释放、因果回收和读者情绪出口 |
| 人物质感（Human Texture） | 给人物加痛苦标签、羞耻标签、关系标签 | 让人物在压力下做选择，并留下关系、情绪、场景和后果 |
| 作品声音 / 作者站位（Work Voice） | 作者跳出来讲道理、老灯说教 | 讲述者稳定存在，但不抢人物方向盘 |
| 角色主动感 / 行动主权（Character Agency） | 主角永远主动出击、永远正确 | 人物基于自己的认知、私心、误判和策略推动局面 |
| 活人浅痕 / 活人漏痕（Live Leakage） | 堆微表情、小动作、人味细节 | 浅、短、带过，不追光，不邀功 |
| 读者代入感 / 读者解压（Reader Immersion） | 解释读者应该怎么想 | 给读者判断、脑补、期待和参与空间 |
| 角色驱动叙事引擎（Agentic Narrative Engine） | 自由多 agent 聊天跑飞 | 有边界的场景仿真，让角色在世界规则和类型承诺内推动事件 |
| 审核门禁（Reviewer Gate） | 只检查格式、爽点数量和模板命中 | 检查作品是否增强了人物、叙述、读者空间和现场感 |

## 6. 项目红线

- 我们不做喂屎机器。
- 我们不满足于低级预制菜。
- 我们要做工程化厨房，目标是稳定逼近现场感。
- 数据只能告诉我们什么容易勾住人，不能告诉我们什么值得写。
- 套路是工具，不是作品的灵魂。
- 钩子负责把人拉进来，真东西负责让人留下来。
- 工程化的目标不是替代真诚，而是降低真诚创作的失败率。
- 每个子项目都要问：它是在增强作品，还是只是在增强操控？
- 不要因为模型“太懂网文”，就允许它生产只有套路、没有真诚的文本。

## 7. 和长期路线图的关系

内容价值观是所有技术模块的上位约束。

人物质感（Human Texture）、作品声音 / 作者站位（Work Voice）、角色主动感 / 行动主权（Character Agency）、活人浅痕 / 活人漏痕（Live Leakage）、读者代入感 / 读者解压（Reader Immersion）、角色驱动叙事引擎（Agentic Narrative Engine）都必须服务这个目标。

如果某个模块只增强套路密度，却不提升人物活性、叙述诚意、读者解压或现场感，应判定为方向风险。

后续所有 MVP 的 acceptance criteria 都应增加一条：是否避免滑向纯套路、纯操控、低质投喂。
