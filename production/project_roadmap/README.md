# Project Long-Term Roadmap v0

本目录用于沉淀项目长期框架、阶段路线图、依赖关系、内容价值观和防跑偏原则。

它不是新的生产链路，不是立即替换当前 `skill-pack` 的方案，也不是一次正文验证或文学定稿。它的作用是把人物质感（Human Texture）、作品声音 / 作者站位（Work Voice）、角色主动感 / 行动主权（Character Agency）、活人浅痕 / 活人漏痕（Live Leakage）、读者代入感 / 读者解压（Reader Immersion）以及更长线的角色驱动叙事引擎（Agentic Narrative Engine）放到同一张项目地图里，回答：

- 现在已经做到哪一步。
- 每个中小项目解决什么问题。
- 它们之间有什么依赖。
- 哪些产物已经完成或可复用。
- 哪些任务应该下一步推进。
- 哪些方向属于长线研究，不应打断当前主线。

## Scope

本目录只新增规划文档。当前版本不修改 `skill-pack/creation_skills/`，不修改 `production/phase8/`，不新增 `approved_patterns`，不生成小说正文，不读取或保存 raw corpus，不启动 Hermes / DeepSeek，也不实际运行 A/B/C 验证。

## Top Constraint

内容价值观是长期路线图的顶层约束。项目定位是工程化厨房，而不是喂屎机器，也不是低级预制菜工厂。工程化要服务人物活性、叙述诚意、读者解压空间和现场感，而不是只服务套路密度和点击操控。

## How to Read

- `content_philosophy.md`：内容价值观与反低质投喂原则，是所有技术模块的上位约束。
- `terminology_policy.md`：术语使用规则，要求中文主名 + 英文括号补充 + 一句话解释。
- `master_framework.md`：总框架与两条主线。
- `roadmap_mermaid.md`：三张 Mermaid 图，用于对齐层级、长线架构和推进顺序。
- `project_dependency_map.md`：子项目依赖和启动条件。
- `current_assets_positioning.md`：现有成果归位表。
- `subproject_breakdown.md`：A-G 子项目拆解。
- `execution_sequence.md`：建议推进节奏。
- `paradigm_shift_agentic_narrative.md`：角色驱动叙事引擎（Agentic Narrative Engine）长线范式。
- `anti_drift_principles.md`：防跑偏原则。
- `next_actions.md`：下一步任务清单。
