# Renderer Skill — Seed-to-Prose 扩写

## 用途

接收一句话章节种子（seed），扩写为 2500-3500 中文字的完整网文章节正文。

## 输入

- `production/renderer_skill_lab/inbox/seeds/chXXX.md`：章节一句话种子
- 本章 Skill：扩写规则与约束

## 输出

- `production/renderer_skill_lab/inbox/expansions/chXXX_v1.md`：纯正文，无标题、无注释

## 扩写硬性规则

### 规则一：事件驱动开场

开章 200 字内必须进入压力/冲突/任务/异常场景。
三种入口：冲突入口、任务入口、异常入口。
禁止开章用大段设定说明、纯环境描写、长篇内心独白。

### 规则二：场景优先于说明

让读者在场景里看见人，而不是听作者解释人。
优先写：当下压力、人物反应、对白错位、动作阻滞、选择后果、留白。

### 规则三：字数 2500-3500 中文字

低于 2500 或超过 3500 为不合格。目标 3000 字左右。

## 严格禁止的 8 种写法

1. 概念 + 破折号解释：「灵根——修仙者的天赋根基——需要检测……」
2. 「不是 A，而是 B」解释架
3. 「认识多年才看得出」类迂回描述
4. 「一毫米表情」伪细腻
5. 动作后立刻翻译心理：「他握紧拳头，心中充满了愤怒」
6. 把潜台词说破
7. 段尾总结中心思想
8. 人物像在完成 prompt 任务

## 禁止项

- 不得引用原文、不得接触原文
- 不得写标题、分析、自评
- 不得解释创作意图
- 不出现工程术语（state_delta, ledger, chapter_card 等）
- 不得模仿具体作者
- 不要写成设定说明、剧情报告、prompt 完成稿
