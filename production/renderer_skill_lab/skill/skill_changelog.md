# Renderer Skill Changelog

## v0.9 — 2026-06-12 — Overcorrection guard + main-image discipline

### 快照说明

本版本在 v0.8 基础上补入“防过度修复”规则。目标是避免 Skill 执行得过实，导致正文变成规则合规但不轻巧的证明型文本：开头旁证过多、配角过早抢镜、制度细节膨胀、荒唐任务被行政流程吞掉、每个错位都补一句机智话。

### 本次变更

| 层级 | 变更 |
|------|------|
| 优先级 | 新增“主画面纪律与防过度修复”层，置于句子自然密度之前。 |
| 不可回退红线 | 新增“主画面不得被证明性细节挤占”。 |
| 不可回退红线 | 新增“强者出场要写压力作用在人物身上，而不是危险结束后人物站着”。 |
| 不可回退红线 | 新增“配角结构功能不得靠新增制度、机构、旧案、档案系统硬撑”。 |
| 不可回退红线 | 新增“荒唐任务不得过早行政化、流程化”。 |
| 不可回退红线 | 新增“结构反差足够强时，不必每处都补聪明话”。 |
| 不可回退红线 | 新增“禁止证明型叙述过量”，防止每一段都在证明场面强、人物强、逻辑合理。 |
| 大场面规则 | 明确大场面开头先建立“地点、压力、主体、主体承压方式、期待/缺席”，旁观者和机制细节后置。 |
| 配角规则 | 明确配角可以有结构功能，但不得偷走第一主画面，也不得为了有用而扩设定。 |
| 荒唐任务规则 | 明确先保留荒唐问题的活性，再进入现实调查；不要一上来变成档案/部门/流程推进。 |
| 证据层 | 新增“对照物最好场景化”，避免参数化比较。 |
| 工作流 | 新增 main-image check，识别哪些细节是在证明规模、哪些是真正在推进主画面。 |

### 回归目标

本版本重点防止以下回退：

- 大场面开头在主角出现前塞入过多旁证；
- 配角、阵法、屏障、数字、机构抢走第一镜头；
- 强者出场只写“事后站着”，没有写压力如何落在他身上；
- 为了让配角有结构功能，临时发明机构、旧案、档案、办公室；
- 荒唐任务过早变成行政流程，喜剧问题被流程吞没；
- 每个反差后都补一句机智话，导致作者设计感过强；
- 每段都像在证明写法合理，而不是自然讲故事。

### 合入原则

本次 Delta 继续采用脱敏抽象表达，不保留任何具体小说名、角色名、任务名、原文句子或私有对照片段。具体案例仍应保留在私有 case notes，不进入公开 Skill。

## v0.8 — 2026-06-12 — Motive economy + register-safe warmth delta

### 快照说明

本版本在 v0.7 的公开通用 Skill 基础上继续收紧：保留荒诞温度，但防止作者旁白解释动机、旧史过重、幽默语域出戏、配角只执行命令、价值反差缺少具体对照物。

### 本次变更

| 层级 | 变更 |
|------|------|
| 不可回退红线 | 新增“行动已经证明动机后，不再旁白解释动机”。 |
| 不可回退红线 | 新增“旧史不得成为装饰性回忆，只保留服务当前反差的最小信息”。 |
| 不可回退红线 | 新增“幽默不得破坏叙述语域，避免用出戏的现代评价词制造活气”。 |
| 句子自然密度 | 强化“作者总结句昂贵”：如果人物行动已经说明动机，不再补总结句。 |
| 人物层 | 强化反应性幽默必须来自错位本身，不能依赖外部段子或语域不合的流行表达。 |
| 配角层 | 强化次要角色的结构功能：除抬位格和见证外，应尽量提供思路支点、限制、资料路径、误判或外部压力。 |
| 证据层 | 新增“价值反差优先使用具体对照物”，避免泛泛说主角资源很多。 |
| 触发规则 | 新增“配角作为思路触发器”规则：一个配角贡献一个实际结构功能即可。 |
| 工作流 | 生成前新增“哪些动机已由行动证明、不应再总结”“旧史最小有效信息”“具体对照物”识别项。 |

### 回归目标

本版本重点防止以下回退：

- 人物已经改变行动后，作者又解释“所以这件事成了线索”；
- 用旧街、旧雨、旧穷困等细节堆怀旧氛围，但不改变当前判断；
- 为了增加活气使用出戏的现代网络评价词；
- 配角只负责站场、失望、领命，不提供任何结构支点；
- 用“主角现在资源很多”这种泛化说法证明迟到资源无用；
- 已经通过具体问答体现动机后，又旁白总结动机；
- 公开 Skill 混入私有案例痕迹。

### 合入原则

本次 Delta 继续采用脱敏抽象表达，不保留任何具体小说名、角色名、任务名、原文句子或私有对照片段。具体案例仍应保留在私有 case notes，不进入公开 Skill。

## v0.7 — 2026-06-12 — Public generic non-regression + temperature delta

### 快照说明

本版本将 Renderer Skill 重构为公开通用提示词资产，并继续补入一组脱敏后的 v0.7 Delta。目标不是记录某一章的具体经验，而是把可复用编辑判断固化为公开规则。

### 本次变更

| 层级 | 变更 |
|------|------|
| 资产边界 | 明确 Skill 不得包含真实小说标题、真实角色名、真实章节情节、原文句子或私有案例结论。 |
| 不可回退红线 | 新增“去 AI 味后不得降温”和“不得过度自证反差”。 |
| 句子自然密度 | 强化作者总结句成本：凡是替读者总结情绪、荒诞、沉重、讽刺的句子，优先改为行动或后果。 |
| 人物层 | 新增“反应性幽默必须来自错位”“旧史必须服务当前反差”“次要角色必须有结构功能”。 |
| 证据层 | 新增“价值判断必须有证据支点”“价值反差优先用层级/对照物证明”。 |
| 触发规则 | 新增“荒唐任务或不可能条件”规则：结尾需要行动方向 + 解法苗头，不能只留下调查或笑点。 |
| 工作流 | 生成前新增“情绪温度、解法苗头、配角结构功能”识别项；生成后新增 public-asset discipline check。 |

### 回归目标

本版本重点防止以下回退：

- 杀掉 AI 修辞后，正文变成低温流程；
- 角色只会核对、追问、安排任务，缺少欲望和荒诞反应；
- 为证明一个反差反复解释三四层；
- 价值判断空口断言，没有可见依据或对照物；
- 旁观者只站着、等着、收命令，没有结构功能；
- 荒唐任务结尾只写“去查”，没有解法苗头；
- 公开 Skill 混入私有案例痕迹。

### 合入原则

本次 Delta 均采用脱敏抽象表达，不保留任何具体小说名、角色名、任务名、原文句子或私有对照片段。具体案例仍应保留在私有 case notes，不进入公开 Skill。
