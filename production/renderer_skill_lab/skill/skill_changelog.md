# Renderer Skill Changelog

## v0.7 — 2026-06-12 — Public generic non-regression + temperature delta

### 快照说明

本版本将 Renderer Skill 重构为公开通用提示词资产，并继续补入一组脱敏后的 v0.7 Delta。目标不是记录某一章的具体经验，而是把可复用编辑判断固化为公开规则。

### 本次变更

| 层级 | 变更 |
|------|------|
| 资产边界 | 明确 Skill 不得包含真实小说标题、真实角色名、真实章节情节、原文句子或私有案例结论。 |
| 不可回退红线 | 新增“去 AI 味后不得降温”和“不得过度自证反差”。 |
| 句子自然密度 | 强化作者总结句成本：凡是替读者总结情绪、荒诞、沉重、讽刺的句子，优先改为行动或后果。 |
| 人物层 | 新增“反应性幽默必须来自错位”“旧史必须服务当前反差”“次要角色必须有结构功能”。 |
| 证据层 | 新增“价值判断必须有证据支点”“价值反差优先用层级/对照物证明”。 |
| 触发规则 | 新增“荒唐任务或不可能条件”规则：结尾需要行动方向 + 解法苗头，不能只留下调查或笑点。 |
| 工作流 | 生成前新增“情绪温度、解法苗头、配角结构功能”识别项；生成后新增 public-asset discipline check。 |

### 回归目标

本版本重点防止以下回退：

- 杀掉 AI 修辞后，正文变成低温流程；
- 角色只会核对、追问、安排任务，缺少欲望和荒诞反应；
- 为证明一个反差反复解释三四层；
- 价值判断空口断言，没有可见依据或对照物；
- 旁观者只站着、等着、收命令，没有结构功能；
- 荒唐任务结尾只写“去查”，没有解法苗头；
- 公开 Skill 混入私有案例痕迹。

### 合入原则

本次 Delta 均采用脱敏抽象表达，不保留任何具体小说名、角色名、任务名、原文句子或私有对照片段。具体案例仍应保留在私有 case notes，不进入公开 Skill。
