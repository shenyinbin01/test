# C3 饭堂 / 矿洞信息露出节点 — 任务 Brief

## 目标

写一个网文片段，约 1200-1800 字。

## 人物

- **苏衍**：新入门杂役弟子，需要发现矿洞和阵眼异常——但不能通过"直接看懂底层源码"的方式。
- **杂役弟子若干**：提供生活场景和信息误差。
- **监工或执事**：代表制度压力（可出现可不出现）。
- **老杂役或矿洞相关人物**：可出现，但不要让他直接完整解释真相。

## 剧情功能（必须覆盖）

1. 苏衍在饭堂或杂役生活场景中发现线索（碎片化的、有人为误差的线索）
2. 线索指向后山矿洞或阵眼异常
3. 苏衍不能直接读懂底层源码——信息必须通过饭堂秩序、杂役制度、旧物、误读、旁人反应、风险代价逐步露出
4. 需要让读者知道：矿洞/阵眼有问题，枯竭可能不是普通事故
5. 苏衍获取信息需要付出代价（时间、人情、制度风险）
6. 结尾留下后续风险或追踪压力

## 写作要求

- 信息不能写成系统公告——不能有"他明白了这是一个分配系统"式的旁白
- 开章即进入场景
- 保留网文推进节奏
- 不使用任何已有章节原文句子
- 不参考 chapter_003_draft.md 的结构和句子
- 不参考 MVP / dry run / regression 的改写文本
