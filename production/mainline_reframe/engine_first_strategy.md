# Engine First Strategy

## 定义

发动机 = 角色主动感 / 行动主权（Character Agency）+ 后果继承账本（Consequence Ledger）。

它解决的不是“句子像不像人”，而是“人物是不是在局面里基于自己的目标、判断、误判、取舍和代价推动下一步”。

## 它不是什么

- 不是让主角永远主动出击。
- 不是让主角永远正确。
- 不是让主角永远强势。
- 不是多写动作。
- 不是多写心理解释。
- 不是自由多 agent 聊天。

## 为什么先造发动机

Human Texture 能让人物、关系和信息不再像功能件。Work Voice 能让叙述者站位更稳定。但如果人物没有行动主权，场景仍然会像作者或系统推着角色走。

角色有行动主权后，作者站位才不用代偿式讲课。角色会因为自己的误判、资源、关系债和私心产生行动，而不是等剧情指令推着他反应。

## 后果继承的作用

后果继承账本保证每章不是一次性爽点。角色的选择应进入下一章状态，表现为：

- 资源变化
- 关系债变化
- 信息状态变化
- 名声或制度位置变化
- 身体或情绪代价
- 下一场景必须处理的摩擦

没有 ledger，所谓代价很容易变成当前段落的气氛，而不是故事的燃料。
