# Post Round 2 Reframe

第二轮 Deep Research 之后，项目主线从“模块逐个增强”重排为“长篇热路径闭环 + 场景发动机 + 受限渲染 + 写回治理”。

## 旧假设调整

| 旧假设 | 新判断 |
|---|---|
| Work Voice 之后继续扩叙述合同即可改善正文 | Work Voice 有价值，但必须收口为 Renderer 合同子模块；它不能替代角色行动主权。 |
| Reader Immersion 可以做 Writer prompt | Reader Immersion 应进入章节卡、读者问题账本和 reviewer gate。 |
| Live Leakage 是独立层 | Live Leakage 是 Human Texture 的微观技法。 |
| Agentic Narrative Engine 是更根本的主线替换 | 它是长线 sandbox，不进入当前热路径。 |
| 完整 IP 宇宙是长篇系统入口 | IP 宇宙是冷层资产，当前只做 `world_slice`。 |
| 一章一句话可以管理章节链 | `chapter_one_sentence` 只是导航锚点，必须升级为 `chapter_card`。 |
| 状态总结越长，连续性越稳 | 连续性来自 `state_delta` 和 ledger reducers，不来自长 summary。 |

## 新执行顺序

1. 冻结 Human Texture v0 基础能力。
2. 完成 Character Agency 和后果继承的 schema / gate。
3. 建立 Longform hot path schema。
4. 定义 Story Orchestrator Lite 和 Renderer Contract。
5. 创建三章连续 MVP 设计包。
6. 人工审查设计包。
7. Codex synthetic dry run，只测 artifact shape 和状态流转。
8. 获批后 Hermes / DeepSeek 跑三章连续小样本。

## 当前阶段边界

本阶段只产出设计包，不生成正文、不修改正式 skill-pack、不调用模型、不接入外部项目、不创建真实长篇大纲。
