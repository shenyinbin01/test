# Layer Reframe

第一轮 Deep Research 建议将原有模块重组为四层，避免把不同类型的问题混在同一 prompt 层。

## 第一层：因果主权层

职责：让人物能自己推动局面，并让选择进入后续状态。

- 角色主动感 / 行动主权
- 后果继承账本
- 关系债账本
- 信息状态变化

失败信号：

- 人物只是在标准反应。
- 冲突只服务设定展示。
- 代价不进入下一章。
- 关系变化没有账本继承。

## 第二层：场景质感层

职责：让场景、人物和信息像发生在人身上，而不是发生在系统面板上。

- 人物质感（Human Texture）
- 活人浅痕作为微观实现
- 信息载体
- 场景阻力

失败信号：

- 信息像公告。
- 场景只是触发器。
- 细节只起装饰作用。
- 人物像功能件。

## 第三层：叙述合同层

职责：让“讲故事的人”稳定存在，并控制作者站位、叙述距离和读者关系。

- 作品声音 / 作者站位（Work Voice）
- 叙述距离
- 读者关系
- 反老练讲课

失败信号：

- 作者跳出来解释意义。
- 叙述者忽远忽近。
- 语言像系统总结。
- 读者被喂结论，没有参与空间。

## 第四层：评价治理层

职责：用 gate 和 rubric 阻止链路滑向顺滑但空心的文本。

- 读者代入感 / 读者解压 rubric
- 反低质投喂门槛
- 模板风险
- Polisher 边界

失败信号：

- Polisher 被要求修复结构性空心。
- 质量判断只看顺滑度。
- 工程化变成套路密度优化。
- Reviewer 无法定位退回层级。
