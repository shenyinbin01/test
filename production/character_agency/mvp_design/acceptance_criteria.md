# Acceptance Criteria

Character Agency MVP 通过标准：

1. `scene_agency_packet.schema.yaml` 可被 YAML 解析。
2. `consequence_ledger.schema.yaml` 可被 YAML 解析。
3. `relationship_debt_ledger.schema.yaml` 可被 YAML 解析。
4. 每个字段都有 description、examples、forbidden_usage。
5. Reviewer gate 明确 `not_polisher_job: true`。
6. 一场景 MVP 能产生 A/B/C 对照设计，不生成正文。
7. C 组理论上必须比 B 组多出“人物选择改变局面”的可审查证据。
8. C 组不允许牺牲网文推进、钩子、规则破局和节奏。
9. 后果必须能进入下一场景或下一章。
10. 失败也必须留下 ledger、gate 与风险记录。
