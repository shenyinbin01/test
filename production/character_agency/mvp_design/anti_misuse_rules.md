# Anti-Misuse Rules

## 禁止误用

1. 不要把 Character Agency 理解成主角永远主动出击。
2. 不要把行动主权写成更多动作戏。
3. 不要把人物误判写成降智。
4. 不要让 Planner 写正文。
5. 不要让 Writer 逐项解释 packet。
6. 不要让 Reviewer 只说“AI 味重”。
7. 不要让 Polisher 修复行动主权缺失。
8. 不要把 consequence ledger 写成抽象“关系变差”。
9. 不要把 relationship debt 变成好感度数字。
10. 不要把多 agent 作为当前 MVP。

## 安全用法

- 每个关键场景只抓一个 active_character。
- 每个 packet 只服务一个具体选择。
- 每个后果必须能在下一场景显影。
- 关系债必须有压力点，而不是只做记录。
