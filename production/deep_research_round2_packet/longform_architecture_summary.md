# Longform Architecture Summary

项目提出的长篇叙事生产链路：

```text
IP 宇宙
  -> 世界 / 子世界
  -> 单书核心故事
  -> 主线 / 分支线
  -> 卷结构
  -> 章节链
  -> 场景 / 冲突单元
  -> 角色主动感发动机
  -> 正文渲染
  -> 状态回写
  -> 下一章继续
```

压缩表达为：

```text
IP 宇宙 -> 单书故事 -> 卷结构 -> 章节链 -> 场景发动机 -> 正文渲染 -> 状态回写
```

## 第二轮需要审计

- 这个链路是否合理。
- 哪一层最容易过度工程化。
- 哪一层最容易造成长篇跑偏。
- 哪些账本是必要的，哪些会增加噪音。
- 最小长篇 MVP 应从哪一层开始。
