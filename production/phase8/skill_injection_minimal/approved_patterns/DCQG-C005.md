# DCQG-C005: 短钩快兑 / 长钩慢兑

## Solves Writing Problem
读者觉得"拖"——钩子开了很多但迟迟不兑现；或者"没悬念"——完全没有阅读期待。

## Mechanism
1. **短钩定义**：本章内或下章即能兑现的小悬念（1~3 章周期）
2. **长钩定义**：跨卷/跨阶段的大悬念（1~3 卷周期）
3. **短钩快兑**：本章开的短钩最晚下章兑现，兑现时升级为更大冲突
4. **长钩慢兑**：长钩在每章/每 2 章有微小推进（不一定是答案，可以是线索/障碍/新问题）
5. **章尾钩子**：每章结尾必须留下一个拉力——短钩或长钩的推进

## How To Use
- **Writer**: 每章结尾必须有 hook
- **Planner**: chapter beat 中标注本章开的钩子（short/long）
- **Reviewer**: 逐章检查 hook_pacing、payoff_visibility
- **Polisher**: 强化章尾钩子的情绪冲击

## Positive Abstract Example
短钩：本章结尾主角在古墓中发现一块刻着自己名字的墓碑——下章开头揭示是预言之墓。
长钩：卷一开头暗示天劫是人为产物——每 5~10 章出现一个碎片线索，直到卷三揭示劫主真身。

## Negative Example
开了 10 个钩子，5 章过去一个都没兑现——读者会忘记原来的悬念，或觉得作者在拖。

## Boundary
- 每章至少 1 个短钩
- 同期进行中的长钩不超过 3 个（超出读者记忆负荷）
- 最长长钩不超过 3 卷未兑现实质性进展
