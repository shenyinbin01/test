# DCQG-C001: 认知优势

## Solves Writing Problem
主角拥有相对于世界、敌人、配角、甚至读者的高维信息优势，但如何不让它变成"主角全知全能"的廉价爽点？

## Mechanism
1. **定义认知差**：主角在某个具体领域比世界多知道一层真相（系统规则、世界机制、上层博弈逻辑）
2. **分层揭示**：部分认知只有主角自己知道（内心优势），部分认知通过行为间接展示给读者
3. **不是全知**：认知优势只覆盖特定领域，主角在其他方面可以无知甚至犯错
4. **不写成旁白**：通过动作、决策、对话中的"反常选择"来展示，而非大段解释

## How To Use
- **Planner**: Bible 中定义 protagonist_cognitive_advantage 字段
- **Writer**: 正文不解释，通过行动展示认知差
- **Reviewer**: 检查是否写成旁白解释
- **Polisher**: 强化认知差带来的读者"恍然"感

## Positive Abstract Example
主角进入一个以"战功积分"决定一切的宗门。所有人都拼命刷分，主角却因为知道积分系统的隐藏规则——"扣分换情报比刷分效率高"——故意违规被扣分去换关键情报。其他角色不解，读者到揭示时恍然。

## Negative Example
主角每章都在内心独白"我早就知道会这样"，但读者从未看到他是如何知道的——认知优势变成了口头解释。

## Boundary
- 每 2~3 章显性使用一次认知优势即可
- 适用于智斗、制度博弈、规则对抗类剧情
- 纯战斗碾压类不需要此技法
