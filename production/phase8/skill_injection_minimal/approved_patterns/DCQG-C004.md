# DCQG-C004: 事件驱动开场

## Solves Writing Problem
每章开头都用大段设定说明、环境描写、内心独白——读者还没进入状态就已经想跳章。

## Mechanism
1. **压力场优先**：开章第一段即进入压力、冲突、任务、或异常状态
2. **三种标准入口**：冲突入口（双方正在对峙/战斗/争论）、任务入口（主角正在执行某个具体行动）、异常入口（世界出现了不该出现的变化）
3. **设定后置**：必要的背景信息穿插在行动中，不前置说明
4. **情绪先行**：让读者先感受到紧张/好奇/愤怒/恐惧，再慢慢解释原因

## How To Use
- **Writer**: 开章 200 字内必须进入压力/冲突/任务/异常场景
- **Planner**: chapter beat 中标注本章开章触发点类型
- **Reviewer**: 检查开章前 200 字是否在"说明"而非"行动"
- **Polisher**: 如开章合格，强化第一句的冲击力

## Positive Abstract Example
开章第一句："刀锋已经贴在后颈上了——如果裁判不喊停，他真的会砍下来。"

## Negative Example
开章前 500 字在介绍这个世界有三界六道、九品丹药、四大宗门——还没出现任何正在发生的事件。

## Boundary
- 第一卷前几章允许少量铺垫，但每章仍需有一个可感知的"当下事件"
- 纯过渡章节（转场/休整）可放宽但不超过 1 章/卷
- 适用于所有网文章节
