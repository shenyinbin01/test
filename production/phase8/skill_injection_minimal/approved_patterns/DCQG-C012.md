# DCQG-C012: 世界观揭秘驱动

## Solves Writing Problem
世界观设定一堆但读者没兴趣读——如何让设定本身成为读者的期待？

## Mechanism
1. **确认好奇心缺口**：先制造谜面（异常现象、矛盾规则、不可解释事件），把世界观真相做成谜底
2. **分阶段揭示**：revelation_phases，每阶段只揭一层，揭完带出更深一层
3. **不是说明书**：每次揭示必须伴随剧情转折，读者因为想知道真相而追读，不是被动接收说明
4. **最后一次揭示必须是故事高潮的一部分**：最大秘密在最高冲突点揭晓

## How To Use
- **Planner**: 卷/阶段设计时标注 revelation_phases，每阶段的核心谜面和谜底
- **Writer**: 每章至少推进一个揭秘点（可以是一小步）
- **Reviewer**: 检查揭秘是否有剧情分量——有没有伴随转折或冲突升级
- **Polisher**: 强化揭秘时刻的情绪冲击

## Positive Abstract Example
卷一谜面："为什么整个修行界没人能突破瓶颈期？" → 在卷一结尾揭示：不是因为天赋，而是有人在天道中设置了年龄锁。揭示同时就是冲突升级——设置锁的人发现主角快突破了。

## Negative Example
开篇前两章把所有世界观设定用系统面板和信息框灌给读者——这是设定说明，不是揭秘驱动。

## Boundary
- 每卷至少一个核心谜面 + 一个核心谜底
- 最长谜面不超过 2 卷（否则读者忘记谜面是什么）
- 适用于有复杂世界观的仙侠/奇幻/科幻

## Original Contamination Guard
- forbidden_elements: []

- de_originalization_method: 谜面和谜底完全原创
