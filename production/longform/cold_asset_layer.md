# Cold Asset Layer

## 为什么 IP 宇宙暂缓

IP 宇宙有价值，但当前不是热路径入口。完整宇宙能帮助跨书复用世界资源、制度、历史、势力和长期冲突，但它不能直接证明三章连续是否稳定。

如果项目先做完整 IP Universe（IP 宇宙），最容易出现三种偏差：

1. 设定堆砌：资料越来越多，章节连续性仍然断。
2. 前期过度设计：系统获得“搭大系统”的安全感，但不产生可验证改善。
3. 资产幻觉：模型以为背景多就等于故事稳，反而把正文写成公告。

## 当前优先级

当前优先做 World Slice（世界切片），而不是完整 IP Universe。世界切片只记录三章连续热路径必须知道的最小世界约束：

- 当前制度压力。
- 当前地点 / 场所规则。
- 当前冲突相关势力。
- 当前资源或身份限制。
- 当前信息揭示边界。

## 何时再考虑完整 IP 宇宙

完整 IP 宇宙应在三章连续 MVP 证明车架有效后再考虑。具体触发条件：

- 三章连续在状态继承、读者问题、关系债和主线稳定上明显优于当前逐章生成。
- `state_delta` 到 ledger reducers 的成本可控。
- `chapter_card` 没有把人物压成执行件。
- Renderer 没有越权补结构。

在这些条件之前，IP 宇宙保持冷层资产状态。
