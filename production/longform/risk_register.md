# Longform Risk Register

| 风险 | 表现 | 原因 | 缓解 |
|---|---|---|---|
| 完整 IP 宇宙拖慢项目 | 资料很多，三章仍断。 | 冷层资产被误当热路径入口。 | 只做 `world_slice`，完整 IP 宇宙暂缓。 |
| schema 官僚化 | 表很完整，正文像工程产物。 | 字段替代了人物选择和阅读活性。 | 三章 MVP 只测必要字段，并设置反低质投喂 gate。 |
| 章节卡过细 | 人物像被卡片操控。 | `chapter_card` 变成硬大纲。 | 保留 `allowed_divergence_band`，让 scene agency 负责场景内选择。 |
| Orchestrator 变隐藏作者 | 它替角色决定策略或情绪。 | 边界管理被误写成剧情生成。 | 明确“不写正文、不替角色做选择、不写内心”。 |
| Renderer 越权 | Renderer 偷偷补因果、关系债或信息状态。 | 末端模型想让文本顺起来。 | 使用 `renderer_contract` 和 blocker。 |
| ledger 成本过高 | 回写、归并和审稿耗时过大。 | 账本过多且粒度过细。 | Delta First，只记录会影响后续选择或读者期待的事实。 |
| 状态正确但无生气 | 连续性提高，阅读仍空。 | 车架压过 Human Texture / Work Voice。 | Renderer 读取质感与叙述合同，但不能改变结构。 |
| 主线稳定但爽感下降 | 系统太保守，钩子和破局变弱。 | 偏航检测过重。 | Reviewer 同时检查 webnovel function 和 hook/payoff balance。 |
| Reader gate 被写成说教 | 读者问题变成作者解释读者该怎么想。 | Reader Immersion 被误当 Writer prompt。 | 只做 `reader_question_ledger` 和 reviewer rubric。 |
| 假账本 | ledger 出现正文没有证据的事实。 | 自动回写幻觉或人为补洞。 | delta 与正文证据不一致时 fail。 |
