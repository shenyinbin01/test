# Book Spine Layer

Book Spine（单书脊梁）是一本文的热启动入口。它不是完整大纲，也不是世界设定库，而是一本书必须兑现的核心承诺。

## 最小字段

| 字段 | 定义 | 失败风险 |
|---|---|---|
| `book_one_sentence` | 一本书的一句话承诺。 | 只能当宣传语，不能约束卷章。 |
| `protagonist_core_drive` | 主角长期驱动力，不等于单章目标。 | 主角被剧情拖着走。 |
| `central_opposition` | 整书级主对抗力量，可以是人物、制度、世界规则或自我误判。 | 卷目标互相松散。 |
| `end_state` | 本书终局方向，说明世界、主角或关系会到达什么状态。 | 章节推进没有终点感。 |
| `genre_promise` | 类型承诺，如升级、破局、复仇、经营、权谋等。 | 网文推进和读者预期失焦。 |
| `emotional_promise` | 情绪承诺，如压抑后翻身、孤立中结盟、信任破裂后重建等。 | 只有机制，没有余味。 |
| `main_transformation` | 主角或关系的核心转变。 | 人物状态不随长篇推进改变。 |
| `forbidden_drift` | 本书绝不能滑向的方向。 | 主线被漂亮支线或临时爽点带偏。 |

## 使用边界

Planner / Orchestrator 可以读取 Book Spine 约束卷卡和章节卡。Writer 不应直接把这些字段解释给读者。Renderer 只能据此保持叙述承诺，不得重写整书方向。
