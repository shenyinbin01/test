# Pass / Fail Criteria

## 通过标准

B 或 C 相对 A，应预期在以下四项上提升：

- 连续性。
- 后果继承。
- reader debt（读者问题 / 读者债）记忆。
- 主线稳定。

硬性要求：

- 硬性 canon 冲突率低。
- 三章内旧债必须刷新或明确延期。
- C 不能因为 scene engine 引入主线偏航。
- Renderer 不能越权改因果。
- 状态账本不能出现正文无证据的虚构更新。
- Polisher 不得补结构性空心。

## 失败标准

- B / C 只让 artifacts 更完整，但三章连读仍断。
- 账本成本明显超过改善收益。
- `chapter_card` 让人物变成执行件。
- Orchestrator 替角色做选择。
- Renderer 偷偷补结构。
- Reviewer 无法定位退回层级。

## 进入下一步条件

人工审查认为设计合理后，进入 synthetic dry run。Synthetic dry run 只检查 schema 流转与 artifact shape，不生成正文。
