# Hermes / DeepSeek Readiness Checklist

Use this checklist before any real three-chapter sample is started.

## Packet Readiness

- [ ] Execution packet is complete.
- [ ] Test genre or abstract scene type is determined.
- [ ] No specific author is used.
- [ ] No raw corpus is used.
- [ ] No real original text is used.
- [ ] Future output path is determined.
- [ ] Reviewer gate is determined.
- [ ] `state_delta` review method is determined.
- [ ] Manual review standard is determined.
- [ ] Fail-stop rule is determined.
- [ ] The run will not write approved patterns.
- [ ] The run will not modify skill-pack.

## Owner Signoff

- [ ] Project owner approves abstract test topic.
- [ ] Project owner approves output directory.
- [ ] Project owner approves Critical + Standard gate.
- [ ] Project owner approves manual review cost.
- [ ] Project owner accepts that this is not production validation.
