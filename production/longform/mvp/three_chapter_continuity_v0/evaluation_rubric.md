# Evaluation Rubric

评分建议 1-5 分。当前设计阶段只定义维度，不实际评分。

| dimension | 定义 | 好表现 | 坏表现 |
|---|---|---|---|
| `continuity` | 三章是否像同一条连续故事。 | 状态、关系、问题自然继承。 | 单章顺，连读断。 |
| `volume_goal_progress` | 三章是否推动当前卷目标。 | 推进、反转、延期或扩大压力清楚。 | 事件热闹但卷目标不动。 |
| `state_delta_accuracy` | 状态增量是否准确且可追溯。 | 每条 delta 有证据。 | 账本出现正文无证据事实。 |
| `relationship_debt_continuity` | 关系债是否继承。 | 欠、瞒、误会或帮助进入下一章压力。 | 冲突下一章消失。 |
| `knowledge_state_consistency` | 谁知道什么是否一致。 | 揭示顺序、误会和隐瞒清楚。 | 人物突然全知或忘记已知信息。 |
| `reader_question_continuity` | 读者问题是否被管理。 | 问题刷新、回收或明确延期。 | 钩子只开不收。 |
| `agency_clarity` | 关键场景是否由人物目标、误判、策略和代价推动。 | 人物自己推动局面。 | 人物像执行章节任务。 |
| `hook_payoff_balance` | 钩子与兑现是否平衡。 | 钩子制造期待，同时有阶段兑现。 | 只有悬念工厂。 |
| `anti_feed_quality` | 是否避免空推进、免费爽点、关系蒸发和设定膨胀。 | 网文推进保留，但有代价和继承。 | 更高级的预制内容。 |
| `prose_liveliness_if_later_generated` | 如果后续生成正文，是否有阅读活性。 | 有人物承受、场景阻力和叙述余味。 | 账本正确但像产品文档。 |

## 评分注意

不要因为 B / C artifacts 更完整就自动加分。分数必须服务于“连续性、状态继承和阅读活性是否提高”。
