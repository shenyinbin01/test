# State Persistence Layer

## Delta First

State Delta（状态增量）是写后唯一增量事实。每章完成后，先记录 `state_delta`，再由 reducers（账本归并器）生成各类账本视图。

不允许 Writer 或 Renderer 直接编辑 ledger（账本）。不允许用越来越长的 summary 承担连续性。

## reducer 输出账本

- 剧情账本：主线、分支、阻塞和下一步。
- 角色状态账本：目标、策略、压力、盲点、疲劳或伤痕。
- 关系债账本：谁欠谁、谁误会谁、谁避开谁、下一次压力点。
- 信息账本：谁知道、误知道、被瞒着、信息可靠度和揭示状态。
- 资源账本：物品、权限、时间、体力、机会、稀缺度。
- 读者问题账本：当前问题、回收窗口、延期理由和新问题。
- 伏笔账本：伏笔、期待窗口、兑现证据和延期理由。
- 卷目标进度账本：本章对卷目标的推进、延迟、反转或代价扩大。

## 验证原则

- `state_delta` 必须能回连到正文证据或事件日志。
- reducer 只能读取 accepted delta。
- 账本视图不能新增 delta 中不存在的事实。
- 如果 delta 与正文证据不一致，必须 fail，而不是让 Polisher 修。
