# 当前生成链路摘要

这是当前项目判断，不是外部研究结论。

## 当前主线

当前主线仍然是基于 skill-pack 的生成链路增强，不是直接替换为角色驱动叙事引擎（Agentic Narrative Engine）。

## 角色与职责

| 角色 | 当前职责 | 边界 |
| --- | --- | --- |
| Hermes | 总调度、任务拆解、验收、状态协调 | 不默认亲自大量写代码；不替项目负责人做最终文学判断 |
| DeepCode | 编码代理，负责脚本、目录、schema、测试、工程报告 | 不负责剧情判断、人物动机判断、正文润色 |
| DeepSeek | 模型层，负责生成大纲、beat、正文草稿、审稿、润色建议等 | 不负责长期记忆 |
| Codex | 工程执行与资料整理，当前任务中只负责生成上传资料包 | 不执行 Deep Research，不联网搜索，不生成正文 |
| ChatGPT / 项目负责人 | 主动决策方、外部审计方、文学判断方 | 决定是否接受阶段成果和长期方向 |
| 规划者（Planner） | 生成 Story Bible、章节大纲、chapter beat、读者期待债规划 | 不写正文、不审稿、不润色、不更新状态 |
| 写作者（Writer） | 根据 Planner 的 beat 和上下文写正文草稿 | 不审稿、不润色、不生成 final、不自由改 beat |
| 审核者（Reviewer） | 多维度审稿，输出结构化报告和修改指令 | 不改正文、不新增剧情、不做润色 |
| 润色者（Polisher） | 在结构通过后做轻量表达增强和去 AI 味修正 | 不救结构、不新增爽点、钩子、主角动机或因果 |

## 状态真源

`.story-system/` 是故事长期状态真源。WPS 是最终输出和人工编辑载体，不是状态数据库。

GitHub 是任务模板、产物、报告和审计包的真源。稳定规则应固化到项目文档，而不是反复通过聊天传递。

## 当前限制

正式 skill-pack 目前包含 Phase 8 的钩子节奏、规则破局、认知优势、模板风险等控制，但当前资料显示人物质感（Human Texture）和作品声音 / 作者站位（Work Voice）尚未作为正式 patch 进入当前 skill-pack 基线。
