# GitHub 读取顺序

本文件用于说明 Deep Research 通过 GitHub URL 阅读资料包时的建议顺序。资料包内容是当前项目判断，不是外部研究结论。

| Order | File | Purpose | Must read | Notes |
| ---: | --- | --- | --- | --- |
| 1 | [DEEP_RESEARCH_ENTRYPOINT.md](DEEP_RESEARCH_ENTRYPOINT.md) | GitHub 唯一入口页，说明读取顺序、术语、任务和禁区 | must_read | 首选给 Deep Research 的 URL |
| 2 | [deep_research_github_prompt.md](deep_research_github_prompt.md) | 只提供一个 GitHub 地址时使用的可复制 prompt | must_read | 项目负责人可直接复制 |
| 3 | [project_summary.md](project_summary.md) | 项目目标、问题意识、路线和顶层原则 | must_read | 先读它建立整体上下文 |
| 4 | [terminology_glossary.md](terminology_glossary.md) | 中文主名优先术语表 | must_read | 避免英文术语误读 |
| 5 | [long_term_framework_summary.md](long_term_framework_summary.md) | 两条主线和长期框架 | must_read | 理解当前链路增强与长线范式 |
| 6 | [content_philosophy_summary.md](content_philosophy_summary.md) | 内容价值观、工程化厨房和反低质投喂 | must_read | 顶层约束 |
| 7 | [key_hypotheses.md](key_hypotheses.md) | 需要 Deep Research 审计的关键假设 | must_read | 研究重点 |
| 8 | [research_questions.md](research_questions.md) | 研究问题清单 | must_read | 直接对应报告问题 |
| 9 | [forbidden_boundaries.md](forbidden_boundaries.md) | 禁止事项和高风险误读 | must_read | 防止任务跑偏 |
| 10 | [requested_outputs.md](requested_outputs.md) | 期望输出结构和证据要求 | must_read | 报告格式依据 |
| 11 | [current_pipeline_summary.md](current_pipeline_summary.md) | 当前多角色生成链路摘要 | recommended | 有助于理解现有工程结构 |
| 12 | [completed_assets_summary.md](completed_assets_summary.md) | 已有资产、缺失路径和 pending 项 | recommended | 有助于判断成熟度 |
| 13 | [source_file_index.md](source_file_index.md) | 资料包参考过的仓库路径索引 | recommended | 仅索引，不复制原文 |
| 14 | [upload_manifest.md](upload_manifest.md) | 手动上传模式下的上传清单 | optional | GitHub 入口模式下可不读 |
| 15 | [README.md](README.md) | 资料包说明和使用方式 | optional | 已被入口页压缩覆盖 |
| 16 | [deep_research_prompt.md](deep_research_prompt.md) | 手动上传资料包时使用的 prompt | optional | GitHub 模式优先使用 `deep_research_github_prompt.md` |
| 17 | [github_reading_order.md](github_reading_order.md) | GitHub 读取顺序和文件用途 | optional | 当前文件，供人工确认 |
| 18 | [packet_status.yaml](packet_status.yaml) | 资料包状态、scope 和输入路径检查 | do_not_upload_manually | Deep Research 通常不必读，除非需要状态元数据 |
