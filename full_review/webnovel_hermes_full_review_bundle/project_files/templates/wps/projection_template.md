# WPS 文档投影模板

## 用途

本章节作品在 WPS / Kdocs 中展示时的格式说明。

## 文档结构

同步到 WPS 的文档应包含以下内容：

1. 封面（第一页）
   - 书名
   - 作者
   - 最新章节

2. 目录（第二页）
   - 已发布章节目录（超链接）

3. 正文
   - 每章单独为一节
   - 章节标题使用 Heading 1 格式
   - 正文宋体 12pt，段落间距 1.5 倍行距

4. 附录（最后一页）
   - 角色卡速查
   - 当前状态概要

## 格式要求

| 元素 | 格式 |
|------|------|
| 书名 | 黑体 28pt，居中 |
| 章节标题 | 黑体 18pt，居中 |
| 正文 | 宋体 12pt，首行缩进 2 字符 |
| 对话 | 宋体 12pt，首行缩进 2 字符，无特殊格式 |
| 分隔符 | `***` 居中 |
| 页边距 | 上下 2.54cm，左右 3.17cm |

## 同步方式

渲染脚本 render_docx.py 生成 .docx 文件后，sync_wps.py 负责：
1. 检查 kdocs-cli 可用性
2. 上传 DOCX 到用户指定的 WPS 文件夹
3. 记录上传结果到 wps/sync_log.jsonl

## 降级策略

如果 WPS 同步失败：
- 保留本地 DOCX 文件
- 记录失败原因
- 不删除、不覆盖已有文档
