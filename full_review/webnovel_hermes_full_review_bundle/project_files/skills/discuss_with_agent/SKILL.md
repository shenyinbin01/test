# Skill 名称: discuss_with_agent

## 所属流程环节
跨环节（用户讨论/修改需求）

## 适用场景
用户对已生成内容提出反馈、修改需求或讨论时。

## 输入
- 用户反馈内容（自然语言）
- 涉及的项目文件路径（用户指定或自动定位）

## 输出

## 示例输入

参见 SKILL.md 中的输入说明

## 示例输出

参见 SKILL.md 中的输出说明
- 修改计划（Markdown，供 apply_discussion_revision 执行）
- 或直接回复用户（如果是讨论而非修改需求）

## 执行步骤
1. 解析用户反馈，识别涉及的文件/环节
2. 如果是修改需求：生成具体的修改计划
3. 如果是讨论：给出分析和建议
4. 将修改计划输出给用户确认

## 质量标准
- 修改计划必须具体到文件路径和改动内容
- 每个改动必须标注影响范围

## 常见失败情况
- 用户需求模糊：给出 2-3 个可能方向让用户选择
- 找不到涉及文件：让用户补充文件路径

## 禁止事项
- 禁止未经确认就直接修改文件
- 禁止猜测用户意图

## 示例输出
"已定位到 MASTER_SETTING.yaml 中林砚的能力限制，您希望修改第 3 条限制还是新增一条？"
