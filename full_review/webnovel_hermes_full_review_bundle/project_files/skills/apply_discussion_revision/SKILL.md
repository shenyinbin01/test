# Skill 名称: apply_discussion_revision

## 所属流程环节
跨环节（修改执行）

## 适用场景
用户确认了修改计划后，执行实际的文件修改。

## 输入
- 用户确认的修改计划
- 涉及的项目文件路径

## 输出

## 示例输入

参见 SKILL.md 中的输入说明

## 示例输出

参见 SKILL.md 中的输出说明
- 修改后的文件
- 修改记录（改动前/后对比）

## 执行步骤
1. 读取修改计划
2. 备份要修改的文件到 backups/ 目录
3. 执行修改
4. 验证修改结果
5. 如果修改涉及的内容已提交 commit，标记为 hotfix

## 质量标准
- 修改前必须备份
- 修改后必须验证文件可解析
- 记录修改时间、内容、涉及文件

## 常见失败情况
- 修改导致 YAML/JSON 格式错误：回退备份
- 修改多文件时部分失败：回退所有已修改文件

## 禁止事项
- 禁止不经用户确认就修改
- 禁止删除备份
- 禁止扩大修改范围

## 示例输出
"已将林砚能力限制第 3 条从'每次使用后头痛'修改为'每次使用后短暂失明，视使用强度延长'"
